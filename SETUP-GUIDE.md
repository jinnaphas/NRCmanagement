# คู่มือ NRC Management System (v3.1)

ระบบบริหารงานคณะกรรมการ NRC — **ไม่ต้องล็อกอิน ไม่ต้องตั้งค่าคลาวด์**
เปิด URL ก็ใช้ได้ทันที เหมาะกับการแชร์ให้กรรมการดู

---

## หลักการทำงาน

| สิ่งของ | เก็บที่ไหน | หมายเหตุ |
|--------|-----------|----------|
| ข้อมูลหลัก (กรรมการ, ประชุม, ค่าตอบแทน ฯลฯ) | ไฟล์ `data.json` ใน repo (เสิร์ฟผ่าน GitHub Pages) | read-only สำหรับผู้ชม |
| รูปกรรมการ & ไฟล์เอกสาร | Google Drive (share link) | วางลิงก์เอง ไม่อัปโหลดผ่านระบบ |

- **กรรมการ/ผู้ชม:** แค่เปิด URL → เห็นข้อมูลล่าสุด คลิกลิงก์เอกสารเปิดได้เลย (ไม่ต้องล็อกอิน)
- **เลขาฯ (Earth):** แก้ไขข้อมูลในระบบ → กด **Export data.json** → นำไฟล์ไปวางทับใน repo เพื่อเผยแพร่

---

## A. เปิด GitHub Pages (ทำครั้งเดียว)

1. repo → **Settings → Pages → Source = GitHub Actions**
2. รอ workflow รันเสร็จ (แท็บ Actions) → ได้ URL เช่น `https://jinnaphas.github.io/nrcmanagement/`
3. แชร์ URL นี้ให้กรรมการ

## B. โครงสร้าง Master List (ทะเบียนกลางกรรมการ)

- เมนู **คณะกรรมการ** = ทะเบียนกลาง เก็บประวัติ/ข้อมูลกรรมการทุกคน **ที่เดียว**
- เพิ่ม/แก้กรรมการ + กำหนดบทบาทในคณะย่อย (NRC/AC/RMC/GCG/ExCo) ที่นี่
- เมนู **Control List** จะดึงรายชื่อจากทะเบียนกลางอัตโนมัติ (คณะกรรมการบริษัท / ชุดย่อย) — แก้ที่เดียว อัปเดตทุกที่

## C. ใส่รูปกรรมการ / ไฟล์เอกสาร (Google Drive share link)

1. อัปโหลดไฟล์/รูปขึ้น Google Drive ของคุณ
2. คลิกขวาที่ไฟล์ → **Share / แชร์** → เปลี่ยนเป็น **Anyone with the link** → **Copy link**
3. นำลิงก์มาวางในช่องที่ระบบเตรียมไว้:
   - **รูปกรรมการ:** หน้าแก้ไขกรรมการ → ช่อง "รูป (Google Drive share link / URL รูป)"
   - **เอกสารคลัง / ไฟล์ประชุม / วาระ:** กดปุ่มแนบ แล้ววางลิงก์ + ตั้งชื่อ
4. กรรมการคลิกชื่อไฟล์ในระบบ → เปิดไฟล์จาก Drive ได้ทันที

## D. เผยแพร่ข้อมูลให้ทีมเห็น (อัปเดต data.json)

1. แก้ไขข้อมูลในระบบจนพอใจ (มุมขวาบนจะขึ้น ✏️ "แก้ไขแล้ว")
2. เมนู **ตั้งค่า → เผยแพร่ข้อมูล** → กด **Export data.json** (ได้ไฟล์ `data.json`)
3. นำไฟล์ `data.json` ไปวางทับในโฟลเดอร์ของ repo (ที่เดียวกับ index.html):
   - วิธีง่าย: GitHub repo → คลิกไฟล์ `data.json` → ปุ่มดินสอ (แก้ไข) / Upload files → วางไฟล์ใหม่ → Commit
   - หรือบอกผม (Claude) ให้ commit ให้ได้
4. GitHub Pages อัปเดตอัตโนมัติใน ~1 นาที → กรรมการเปิด URL เห็นข้อมูลใหม่

> 💡 ถ้ายังไม่มีไฟล์ `data.json` ในระบบ จะใช้ข้อมูลตัวอย่างที่ฝังไว้ในแอปก่อน — Export ครั้งแรกแล้ว commit จะกลายเป็นข้อมูลจริง

---

## ปุ่มที่เกี่ยวข้อง (เมนู ตั้งค่า → เผยแพร่ข้อมูล)

| ปุ่ม | ทำอะไร |
|------|--------|
| **Export data.json** | ดาวน์โหลดข้อมูลปัจจุบันเป็นไฟล์ data.json (เอาไป commit) |
| **โหลด data.json เข้าระบบ** | นำไฟล์ data.json จากเครื่องมาแก้ไขต่อ |
| **โหลดใหม่จากเว็บ** | ดึง data.json ล่าสุดจาก URL มาแสดง |

## ข้อควรระวัง

- ข้อมูลที่กำลังแก้ไขอยู่ใน **หน่วยความจำของหน้าเว็บ** เท่านั้น — **กด Export ก่อนปิดหน้าทุกครั้ง** (ระบบจะเตือนถ้ามีข้อมูลค้าง)
- รูป/ไฟล์ต้องตั้งสิทธิ์ Drive เป็น **Anyone with the link** ไม่งั้นกรรมการจะเปิดไม่ได้
